# Journal AI - Technical Test

Ce journal sert a tracer **toutes tes interactions avec l'IA** pendant l'exercice.
L'objectif est de comprendre ta methode de travail, pas de juger la quantite.

---

## Structure

```
ai-journal/
├── exercice-0-onboarding/   # Onboarding & Outillage
├── exercice-1-statut/       # Statut abonnement
├── exercice-2-modal/        # Modal changement d'offre
├── exercice-3-erreurs/      # Gestion des erreurs
└── retrospective.md         # Bilan final
```

Chaque exercice contient :
- `README.md` : objectif, fichiers crees, reflexions
- `prompts.md` : tous les prompts utilises (sans limite)

---

## Comment utiliser ce journal

1. **Avant chaque exercice** : note ton objectif dans le README de l'exercice
2. **Pendant** : documente chaque prompt dans `prompts.md`
3. **Apres** : complete les reflexions dans le README
4. **A la fin** : remplis `retrospective.md`

---

## Ce qu'on juge / ne juge pas

**On ne juge pas :**
- Le nombre de prompts
- Le fait d'avoir besoin d'iterer
- Le temps passe

**On juge :**
- La qualite des prompts (sont-ils clairs, precis, contextualises ?)
- La capacite a iterer efficacement
- La comprehension de ce que l'IA genere
- La capacite a outiller pour les autres
